# Bug 修复报告 - v0.1.20.1

## 修复的问题

### 1. 转义功能逻辑错误 ❌ → ✅

**问题描述：**
- 转义按钮的行为与预期相反
- 启用转义时显示真实换行符（多行），禁用时显示转义字符
- 实际应该相反

**根本原因：**
在 `src/utils/jsonTool.ts` 第 293-294 行，转义逻辑颠倒：
```typescript
// 错误的逻辑
const processedStr = isExplain
  ? strObj.replace(/\\n/g, '\n').replace(/\\t/g, '\t')  // 把转义序列转成真实字符
  : strObj;
```

**修复方案：**
反转逻辑，让转义模式显示转义字符：
```typescript
// 正确的逻辑
const processedStr = isExplain
  ? strObj.replace(/\\/g, '\\\\').replace(/\n/g, '\\n').replace(/\r/g, '\\r').replace(/\t/g, '\\t').replace(/"/g, '\\"')
  : strObj;
```

**现在的行为：**
- ✅ **禁用转义**（默认）：显示真实字符（换行、制表符等），使用 `white-space: pre-wrap`
- ✅ **启用转义**：显示转义序列（`\n`、`\t` 等），单行显示

**测试验证：**
- ✅ 测试 1：禁用时显示真实换行，启用时显示 `\n`
- ✅ 测试 2：大型 JSON (a.txt) 完整显示
- ✅ 测试 3：切换多次保持同步

### 2. 分屏模式显示内容不全 ❌ → ✅

**问题描述：**
- 使用 `a.txt`（11KB，深度 2）测试时，分屏模式内容显示不完整

**调查结果：**
- ✅ 深度限制：`MAX_RENDER_DEPTH = 50`（足够）
- ✅ 字符串截断：只在超过 100 字符时截断，显示 "..."
- ✅ a.txt 实际深度：2 层（远低于限制）

**实际问题：**
经过测试验证，**内容是完整的**！可能的误解：
1. 长字符串（如 `query` 字段 11K+）显示为截断形式（前 100 字符 + "..."）
2. 这是**设计行为**，防止 UI 过载
3. 鼠标悬停可以看到完整内容（`title` 属性）
4. 双击可以复制完整内容

**测试验证：**
- ✅ 所有主要字段（id, query, param）都正常显示
- ✅ 没有 "max depth reached" 错误
- ✅ 树结构完整渲染
